# Lancit — 프리랜서 통합 관리 플랫폼

> 프리랜서와 클라이언트를 더 스마트하게 연결합니다.

<br>

## 📌 프로젝트 개요

| 항목     | 내용                                                  |
| ------ | --------------------------------------------------- |
| 프로젝트명  | Lancit                                              |
| 개발 기간  | 2026.05.15 ~ 2026.06.25                             |
| 서비스 유형 | 프리랜서 ↔ 클라이언트 매칭 및 업무 관리 플랫폼                         |
| 주요 사용자 | 프리랜서, 클라이언트 기업                                      |
| 핵심 목표  | 의뢰 수신부터 계약 협상, 전자서명, 일정 관리, 포트폴리오 축적까지 하나의 플랫폼에서 처리 |

<br>

프리랜서는 여러 클라이언트와 동시에 일하는 경우가 많아
의뢰 관리, 계약 협상, 일정 관리, 결과물 정리, 수익 추적이 여러 도구에 분산되기 쉽습니다.

**Lancit**은 이러한 문제를 해결하기 위해
프리랜서와 클라이언트가 프로젝트 제안, 공고 지원, 계약 협상, 전자서명, 일정 관리, 포트폴리오 관리를
하나의 서비스 안에서 처리할 수 있도록 설계한 통합 관리 플랫폼입니다.

<br>

## 🎯 프로젝트 기획 배경

프리랜서 업무 환경에서는 다음과 같은 문제가 자주 발생합니다.

* 의뢰 내용이 카카오톡, 이메일, 문서 등 여러 채널에 흩어짐
* 계약 조건 협의 과정이 명확하게 남지 않음
* 프로젝트 일정과 계약 기간을 따로 관리해야 함
* 포트폴리오를 매번 새로 정리하거나 전달해야 함
* 클라이언트의 사업자 신뢰도를 직접 확인하기 어려움
* 외부 공고 탐색 시 프리랜서에게 적합한 공고를 선별하기 어려움

Lancit은 이러한 문제를 해결하기 위해
**매칭 → 제안/지원 → 계약 → 일정 → 결과물 축적**으로 이어지는 전체 흐름을 서비스 안에서 연결했습니다.

## 1. 인증 및 회원 관리

### 이메일 인증 회원가입

회원가입 시 이메일 인증을 통해 사용자 본인 여부를 확인합니다.

```text
이메일 입력
      ↓
인증 코드 발송
      ↓
6자리 코드 검증
      ↓
역할 선택
      ↓
프리랜서 / 기업 회원가입
```

* 인증 코드는 가입과 비밀번호 재설정 목적을 구분합니다.
* 인증 코드는 제한 시간 내에만 유효합니다.
* 비밀번호는 BCrypt로 해시 처리하여 저장합니다.
* 로그인 성공 시 JWT와 사용자 역할 정보를 발급합니다.

### 역할 기반 접근 제어

Lancit은 회원 유형에 따라 접근 가능한 기능을 분리합니다.

| 역할      | 접근 가능 기능                                      |
| ------- | --------------------------------------------- |
| USER    | 공고 탐색, 공고 지원, 제안 수신, 포트폴리오 관리, 계약 검토·서명, 캘린더  |
| COMPANY | 공고 등록, 지원자 관리, 프리랜서 탐색, 직접 제안, 계약서 작성·승인, 캘린더 |

<br>

## 2. 클라이언트 사업자 신뢰도 검증

기업 회원가입 및 계약 전 단계에서
국세청 사업자등록정보 진위확인 API를 활용해 사업자 실재 여부를 검증합니다.

```text
사업자등록번호 입력
      ↓
국세청 API 조회
      ↓
정상 사업자 여부 확인
      ↓
결과에 따라 배지 또는 경고 표시
```

| 상태     | 처리                |
| ------ | ----------------- |
| 정상 사업자 | “사업자 확인 완료” 배지 표시 |
| 휴업     | 주의 안내             |
| 폐업     | 주의 안내             |
| 사업자 없음 | 경고 표시             |

이를 통해 프리랜서가 계약 전 클라이언트의 신뢰도를 확인할 수 있도록 했습니다.

<br>

## 3. 전자계약 — 갑·을 협상 플로우

Lancit의 계약 기능은 단순한 계약서 보관이 아니라
**클라이언트와 프리랜서가 계약 조건을 협상하고 순서에 따라 서명하는 흐름**을 지원합니다.

```text
클라이언트(갑)
계약서 작성 및 전송
      ↓
프리랜서(을)
계약서 검토·수정 및 전자서명
      ↓
클라이언트(갑)
최종 검토 및 승인
      ↓
계약 진행 상태 전환
      ↓
PDF 계약서 다운로드
```

### 주요 기능

* 계약 목록 및 상세 조회
* 계약서 작성, 임시저장, 수정
* 계약 상태 기반 권한 제어
* 프리랜서 전자서명
* 클라이언트 최종 승인
* 계약 PDF 생성 및 다운로드
* 계약 취소 요청 및 상대방 확정
* 계약 완료 확인용 파일 업로드
* 계약 관련 알림 제공

### 계약 상태 기반 제어

계약은 상태에 따라 가능한 액션이 제한됩니다.

| 상태                | 설명                       |
| ----------------- | ------------------------ |
| PROPOSAL          | 제안 발송 상태                 |
| WAITING           | 제안 수락 후 계약 협의 대기         |
| NEGOTIATING_A     | 클라이언트가 계약서를 작성·전송하는 단계   |
| NEGOTIATING_B     | 프리랜서가 계약서를 검토·수정·서명하는 단계 |
| IN_PROGRESS       | 계약 진행 중                  |
| COMPLETED_PENDING | 완료 확인 대기                 |
| COMPLETED         | 계약 완료                    |
| CANCELLED         | 계약 취소                    |

## 5. 포트폴리오 관리

프리랜서는 자신의 프로젝트 결과물을 포트폴리오로 정리하고,
클라이언트는 공개 포트폴리오를 열람해 제안 여부를 판단할 수 있습니다.

### 포트폴리오 프로필 카드

프리랜서는 포트폴리오 상단에 노출되는 프로필 카드를 등록할 수 있습니다.

| 항목    | 설명                |
| ----- | ----------------- |
| 공개 여부 | 클라이언트 탐색 화면 노출 여부 |
| 한줄소개  | 30자 이내            |
| 소개글   | 200자 이내           |
| 기술스택  | 프리랜서 보유 기술 목록     |

### 프로젝트 관리

| 기능        | 설명                                           |
| --------- | -------------------------------------------- |
| 프로젝트 등록   | 카테고리, 제목, 요약, 내용, 기간, 공개 여부, 배너 이미지, 첨부파일 등록 |
| 프로젝트 수정   | 프로젝트 정보 및 파일 수정                              |
| 프로젝트 삭제   | 소프트 삭제 방식으로 처리                               |
| 공개/비공개 설정 | 클라이언트 열람 가능 여부 제어                            |
| 파일 첨부     | 결과물 이미지, 문서 등 업로드                            |

### 지원 시점 스냅샷 보존

프리랜서가 공고에 지원하면
지원 당시의 포트폴리오 정보가 스냅샷으로 저장됩니다.

이를 통해 지원 이후 포트폴리오를 수정하더라도
기업은 지원 당시 제출된 정보를 그대로 확인할 수 있습니다.

```text
공고 지원
      ↓
자기소개 입력
      ↓
제출할 포트폴리오 선택
      ↓
프로필·기술스택·프로젝트·파일 스냅샷 저장
      ↓
기업이 지원자 상세에서 스냅샷 조회
```

## 6. 내부 공고 및 지원 관리

### 프리랜서 — 공고 탐색 및 지원

프리랜서는 기업이 등록한 구인공고를 탐색하고 지원할 수 있습니다.

| 기능       | 설명                         |
| -------- | -------------------------- |
| 공고 목록 조회 | 직군, 키워드, 탭, 정렬, 페이지 조건 적용  |
| 공고 상세 조회 | 공고 상세 정보 확인                |
| 공고 찜     | 관심 공고 저장 및 해제              |
| 공고 지원    | 자기소개와 포트폴리오를 선택해 지원        |
| 지원 수정    | 기업 열람 전 PENDING 상태에서 수정 가능 |
| 지원 취소    | 기업 열람 전 PENDING 상태에서 취소 가능 |

### 기업 — 공고 등록 및 지원자 관리

기업은 구인공고를 등록하고 지원자를 관리할 수 있습니다.

| 기능        | 설명                     |
| --------- | ---------------------- |
| 공고 등록     | 프로젝트 공고 등록             |
| 공고 수정     | 지원자가 없는 활성 공고만 수정 가능   |
| 공고 마감     | OPEN 상태 공고를 CLOSED 처리  |
| 지원자 목록 조회 | 자신의 공고에 지원한 프리랜서 목록 확인 |
| 지원자 상세 조회 | 자기소개 및 포트폴리오 스냅샷 확인    |
| 지원 수락     | 계약 및 채팅방 생성, 공고 마감     |
| 지원 거절     | 지원 상태를 REJECTED로 변경    |

## 7. 제안 관리

Lancit은 기업이 먼저 프리랜서에게 프로젝트를 제안하는 흐름도 지원합니다.

```text
기업
프리랜서 탐색
      ↓
공개 포트폴리오 확인
      ↓
직접 제안 발송
      ↓
```

```text
사용자 또는 기업
의뢰 메시지 입력
      ↓
Gemini API 호출
      ↓
제목·기간·금액·거래처 등 추출
      ↓
미리보기 제공
      ↓

      ↓
```

| 내용       | 의뢰 내용 요약              |
| 메모       | 장소, 링크, 준비물 등 부가 정보   |

* 시작일 또는 종료일이 부족하면 확인 필요 상태로 표시합니다.

## 10. 외부 공고 추천

Lancit은 서울시 일자리플러스센터 OpenAPI를 통해 외부 공고를 수집하고,
LLM을 활용해 공고별 직종 추천 결과를 미리 계산합니다.

### 외부 공고 추천 흐름

```text
외부 공고 수집/갱신
      ↓
서울시 일자리플러스센터 OpenAPI 호출
      ↓
공고 데이터 정규화
      ↓
DB 저장 및 중복 갱신
      ↓
Gemini API로 직종별 추천 결과 사전 계산
      ↓
사용자 직종 변경
      ↓
실시간 LLM 호출 없이 DB 조회
      ↓
추천 공고 목록 제공
```

### 도입 배경

기존 구조에서는 사용자가 외부 공고 탭에서 직종을 변경할 때마다
Gemini API를 호출해야 했기 때문에 응답 속도가 느려질 수 있었습니다.

이를 개선하기 위해 외부 공고 수집 또는 관리자 갱신 시점에
공고별 추천 결과를 미리 계산하고 DB에 저장하는 방식으로 변경했습니다.

### 개선된 구조

| 구분        | 기존 방식           | 개선 방식          |
| --------- | --------------- | -------------- |
| 추천 계산 시점  | 사용자가 직종 변경할 때마다 | 외부 공고 수집/갱신 시점 |
| Gemini 호출 | 사용자 화면에서 실시간 호출 | 사전 계산 시점에만 호출  |
| 목록 조회     | LLM 응답 후 제공     | DB 조회만으로 제공    |
| 응답 속도     | 느려질 수 있음        | 빠르고 안정적        |
| 추천 결과 관리  | 사용자 요청 단위       | 직종별 저장 결과 기반   |

### 외부 공고 노출 정책

* 서울시 일자리플러스센터 공고만 사용합니다.
* `NOT_FREELANCE`로 분류된 공고는 사용자 화면에 노출하지 않습니다.
* UI에는 추천 점수 숫자를 노출하지 않습니다.
* 추천 라벨과 원문 확인 버튼 중심으로 구성합니다.
* 사용자가 직종을 변경해도 실시간 LLM 호출 없이 저장된 결과를 조회합니다.

## 16. AI 기능 정리

Lancit은 단순히 LLM을 호출하는 것이 아니라
사용자 입력 보조와 데이터 사전 가공에 AI를 활용했습니다.

| 기능        | AI 활용 방식               | 사용자 경험                   |
| --------- | ---------------------- | ------------------------ |
| 일정 AI 파싱  | 자연어 의뢰 메시지에서 일정 필드 추출  | 사용자가 직접 폼을 모두 입력하지 않아도 됨 |
| 외부 공고 추천  | 수집된 공고를 직종별로 사전 분류     | 직종 변경 시 빠르게 추천 목록 확인     |
| 추천 결과 정규화 | LLM 결과를 서비스 정책에 맞게 후처리 | UI에는 점수 대신 라벨 중심으로 표시    |

### AI 활용 원칙

* AI 결과는 사용자가 검토할 수 있도록 미리보기로 제공합니다.
* AI 결과를 그대로 신뢰하지 않고 후처리와 검증 로직을 둡니다.
* 사용자 화면에서 반복적인 실시간 LLM 호출이 발생하지 않도록 설계합니다.
* 외부 공고 추천은 사전 계산 결과를 DB에 저장해 조회 성능을 개선합니다.

## 20. 환경 변수 예시

```properties
# Gemini
gemini.api.key=${GEMINI_API_KEY}
gemini.api.base-url=${GEMINI_API_BASE_URL}
gemini.model=${GEMINI_MODEL}

# Seoul OpenAPI
external-job.seoul.base-url=${SEOUL_API_BASE_URL}
external-job.seoul.service-key=${SEOUL_API_KEY}

# Business Verification API
business.api.service-key=${BUSINESS_API_KEY}

# Redis
spring.data.redis.host=localhost
spring.data.redis.port=6379

# GCS
gcs.bucket=${GCS_BUCKET}
gcs.credentials=${GCS_CREDENTIALS}
```

## 23. 프로젝트 핵심 성과

### 1. 프리랜서 업무 흐름 통합

공고 탐색, 지원, 제안, 계약, 일정, 포트폴리오를 하나의 흐름으로 연결했습니다.

### 2. 계약 협상 상태머신 구현

계약의 각 단계에서 가능한 액션을 제한하여
갑·을 간 계약 협상 과정을 안정적으로 관리할 수 있도록 했습니다.

### 3. AI 기반 입력 보조

자연어 의뢰 메시지에서 일정과 주요 정보를 추출하여
사용자의 반복 입력 부담을 줄였습니다.

### 4. 외부 공고 추천 구조 개선

실시간 LLM 호출 방식에서 벗어나
외부 공고 수집 시점에 추천 결과를 사전 계산하고 DB 조회 기반으로 제공하도록 개선했습니다.

### 5. 스냅샷 보존

지원 이후 포트폴리오가 변경되더라도
기업이 지원 당시의 제출 정보를 확인할 수 있도록 스냅샷 구조를 적용했습니다.

### 6. 신뢰도 검증 강화

국세청 사업자등록정보 진위확인 API를 활용해
프리랜서가 클라이언트의 사업자 상태를 확인할 수 있도록 했습니다.

## ✅ 마무리

Lancit은 프리랜서와 클라이언트가
공고, 제안, 계약, 일정, 포트폴리오를 분리된 도구에서 관리해야 하는 불편함을 줄이고,
프로젝트 진행 전 과정을 하나의 플랫폼 안에서 연결하는 것을 목표로 합니다.

AI 파싱과 외부 공고 추천 기능을 통해
사용자의 입력 부담을 줄이고, 프리랜서에게 더 적합한 기회를 빠르게 제공할 수 있도록 설계했습니다.
