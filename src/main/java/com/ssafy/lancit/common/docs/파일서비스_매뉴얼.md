# FileService 사용 매뉴얼

> GCS 업로드, Redis Signed URL 캐싱, DB 저장이 전부 자동으로 처리됩니다.

---

## 1. INSERT (업로드)

| 상황 | 호출 메서드 | 파라미터 | 호출하는 쪽 |
|---|---|---|---|
| 단건 업로드 | `fileService.upload(List.of(file), parentType, parentId, email, role)` | PROFILE → parentId=null / 나머지 → 해당 도메인 ID | - |
| 다건 업로드 | `fileService.upload(files, parentType, parentId, email, role)` | 위와 동일 | - |

**반환값:** `List<FileDTO>`

---

## 2. READ (조회)

| 상황 | 호출 메서드 | 파라미터 | 비고 |
|---|---|---|---|
| fileId 로 단건 조회 | `fileService.findById(fileId)` | fileId | 없으면 FILE_NOT_FOUND 자동 throw |
| 부모 기준 전체 조회 | `fileService.findByParent(parentType, parentId)` | parentType, parentId | - |
| 이미지 렌더링용 URL 조회 | `fileService.getSignedUrl(fileId)` | fileId | Redis 캐시 자동 적용 (TTL 6일) |

> Signed URL 은 프로필 사진, 포트폴리오 배너에서만 사용
> 프론트에서는 `GET /api/files/{fileId}/url` 호출하면 됨 

---

## 3. DELETE (삭제)

| 상황 | 호출 메서드 | 파라미터 | 호출하는 쪽 |
|---|---|---|---|
| 파일 단건 삭제 | `fileService.delete(fileId)` | fileId | @OwnerCheck 자동 적용 (별도 추가 불필요) |
| 부모 기준 전체 삭제 | `fileService.deleteByParent(parentType, parentId)` | parentType, parentId | 호출하는 서비스 메서드에 `@OwnerCheck` + `@Transactional` 필수 |

> ⚠️ `GcsService.deleteByPath()` 직접 호출 금지 (업로드 롤백 전용)

---

## 4. UPDATE (파일 교체)

> 파일 수정 API 없음 → **삭제 후 재업로드** 가 표준 패턴

| 상황 | 순서 | 호출하는 쪽 |
|---|---|---|
| 단건 교체 (프로필 사진 등) | 1. `fileService.upload(List.of(newFile), parentType, parentId, email, role)` → newFileId 받기 | `@Transactional` 필수 |
| | 2. `도메인Mapper.update()` → newFileId 로 교체 | |
| | 3. `fileService.delete(oldFileId)` → 기존 파일 삭제 | |
| 다건 교체 (포트폴리오 결과물 등) | 1. `fileService.deleteByParent(parentType, parentId)` → 기존 전부 삭제 | `@OwnerCheck` + `@Transactional` 필수 |
| | 2. `fileService.upload(newFiles, parentType, parentId, email, role)` → 새 파일 업로드 | |

> ⚠️ `GcsService.deleteByPath()` 직접 호출 금지

---

## 5. 예외 목록

| 예외 | 발생 상황 |
|---|---|
| `FILE_NOT_FOUND` | 존재하지 않는 fileId 조회 시 |
| `FILE_UPLOAD_FAILED` | GCS 업로드 또는 DB INSERT 실패 시 |
| `FORBIDDEN` | 타인 파일 삭제 시도 시 |
| `NOT_FOUND` | `@OwnerCheck` 에서 존재하지 않는 fileId 접근 시 |

---

## 6. GCS 폴더 매핑

> `FileParentType` enum 값으로 전달하면 GCS 폴더 자동 매핑됨 (별도 처리 불필요)

| ParentType | GCS 경로 |
|---|---|
| PROFILE | `profile/` |
| PORTFOLIO_BANNER | `portfolio/banner/` |
| PORTFOLIO_FILE | `portfolio/file/` |
| CONTRACT | `contract/` |
| CHAT | `chat/` |
