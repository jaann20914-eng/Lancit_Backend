# FileService 사용 매뉴얼

> GCS 업로드, Redis Signed URL 캐싱, DB 저장이 전부 자동으로 처리됩니다.
> ⚠️ parentId 유효성 검증은 호출하는 쪽 서비스에서 반드시 처리해야 합니다.

---

## 1. INSERT (업로드)

| 상황 | 호출 메서드 | 파라미터 | 호출하는 쪽 필수 |
|---|---|---|---|
| 단건 업로드 | `fileService.upload(List.of(file), parentType, parentId, email, role)` | PROFILE → parentId=null / 나머지 → 해당 도메인 ID | `@Transactional` |
| 다건 업로드 | `fileService.upload(files, parentType, parentId, email, role)` | 위와 동일 | `@Transactional` |

**반환값:** `List<FileDTO>` → `result.get(0).getFileId()` 로 fileId 꺼내기

> ⚠️ PROFILE 이 아닌 경우 parentId 가 실제 존재하는지 호출 전에 검증 필요

---

## 2. READ (조회 / 렌더링 / 다운로드)

| 상황 | 호출 메서드 | 파라미터 | 호출하는 쪽 필수 |
|---|---|---|---|
| fileId 로 단건 조회 | `fileService.findById(fileId)` | fileId | 없음 (없으면 FILE_NOT_FOUND 자동 throw) |
| 부모 기준 전체 조회 | `fileService.findByParent(parentType, parentId)` | parentType, parentId | 없음 |
| 이미지 렌더링용 URL | `fileService.getSignedUrl(fileId)` | fileId | 없음 (Redis 캐시 자동 적용 TTL 6일) |
| 파일 다운로드용 URL | `fileService.getDownloadUrl(fileId)` | fileId | 없음 (Redis 캐싱 안 함, 매번 새로 발급) |

### 렌더링 vs 다운로드 차이

| | 렌더링 (`getSignedUrl`) | 다운로드 (`getDownloadUrl`) |
|---|---|---|
| 용도 | 프로필 사진, 포트폴리오 배너 이미지 표시 | 포트폴리오 결과물, 채팅 첨부파일 다운로드 |
| TTL | 7일 | 1시간 |
| Redis 캐싱 | ✅ (TTL 6일) | ❌ (매번 새로 발급) |
| ContentDisposition | inline (브라우저에서 열림) | attachment (바로 다운로드) |
| 프론트 호출 | `GET /api/files/{fileId}/url` | `GET /api/files/{fileId}/download` |

---

## 3. DELETE (삭제)

| 상황 | 호출 메서드 | 파라미터 | 호출하는 쪽 필수 |
|---|---|---|---|
| 파일 단건 삭제 | `fileService.delete(fileId)` | fileId | 없음 (@OwnerCheck + @Transactional 내부 자동 적용) |
| 부모 기준 전체 삭제 | `fileService.deleteByParent(parentType, parentId)` | parentType, parentId | `@OwnerCheck` + `@Transactional` |

> ⚠️ `GcsService.deleteByPath()` 직접 호출 금지 (업로드 롤백 전용)
> ⚠️ `deleteByParent()` 는 @OwnerCheck 없음 → 호출하는 서비스 메서드에서 소유자 검증 필수

---

## 4. UPDATE (파일 교체)

> 파일 수정 API 없음 → **삭제 후 재업로드** 가 표준 패턴

### 단건 교체 (프로필 사진 등)

| 순서 | 호출 | 호출하는 쪽 필수 |
|---|---|---|
| 1 | `fileService.upload(List.of(newFile), parentType, parentId, email, role)` → newFileId 받기 | `@Transactional` |
| 2 | `도메인Mapper.update()` → newFileId 로 교체 | |
| 3 | `fileService.delete(oldFileId)` → 기존 파일 삭제 | |

### 다건 교체 (포트폴리오 결과물 등)

| 순서 | 호출 | 호출하는 쪽 필수 |
|---|---|---|
| 1 | `fileService.deleteByParent(parentType, parentId)` → 기존 전부 삭제 | `@OwnerCheck` + `@Transactional` |
| 2 | `fileService.upload(newFiles, parentType, parentId, email, role)` → 새 파일 업로드 | |

> ⚠️ `GcsService.deleteByPath()` 직접 호출 금지

---

## 5. 예외 목록

| 예외 | 발생 상황 |
|---|---|
| `FILE_NOT_FOUND` | 존재하지 않는 fileId 조회 시 |
| `FILE_UPLOAD_FAILED` | GCS 업로드 또는 DB INSERT 실패 시 |
| `FORBIDDEN` | 타인 파일 삭제 시도 시 |
| `NOT_FOUND` | `@OwnerCheck` 에서 존재하지 않는 fileId 접근 시 |

---

## 6. GCS 폴더 매핑

> `FileParentType` enum 값으로 전달하면 GCS 폴더 자동 매핑됨 (별도 처리 불필요)

| ParentType | GCS 경로 | 렌더링/다운로드 |
|---|---|---|
| PROFILE | `profile/` | 렌더링 (`getSignedUrl`) |
| PORTFOLIO_PROFILE | `portfolio/profile/` | 렌더링 (`getSignedUrl`) |
| APPLICATION_PROFILE | `application/profile/` | 렌더링 (`getSignedUrl`) |
| PORTFOLIO_BANNER | `portfolio/banner/` | 렌더링 (`getSignedUrl`) |
| PORTFOLIO_FILE | `portfolio/file/` | 다운로드 (`getDownloadUrl`) |
| RECRUITMENT_IMAGE | `recruitment/image/` | 렌더링 (`getSignedUrl`) |
| CONTRACT | `contract/` | 다운로드 (`getDownloadUrl`) |
| CHAT | `chat/` | 다운로드 (`getDownloadUrl`) |
